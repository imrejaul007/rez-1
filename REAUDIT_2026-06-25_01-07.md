## 2. Key Flows Analysis

### 2.1 Payment Flow

#### 2.1.1 Razorpay Integration

**Backend (razorpayService.ts):**
```typescript
- createRazorpayOrder(amount, receipt, notes)
- verifyRazorpaySignature(orderId, paymentId, signature)
- fetchPaymentDetails(paymentId)
- createRefund(paymentId, amount?)
- createRazorpayPayout(params) // For cashback
```

**Security Features:**
- ✅ HMAC-SHA256 signature verification with timing-safe comparison
- ✅ Idempotency keys for order creation
- ✅ Amount validation (minimum 1 INR)
- ✅ Timeout handling (10s default)
- ✅ Webhook payload validation

**Frontend (useCheckout.ts):**
```typescript
handleRazorpayPayment()
  → createRazorpayPayment() → razorpayApi
  → onSuccess: ordersService.createOrder()
  → Navigate to /payment-success
```

#### 2.1.2 Stripe Integration

**Frontend (paymentService.ts):**
- `createPaymentIntent()` → `/wallet/stripe/create-payment-intent`
- `confirmPayment()` → `/wallet/confirm-payment`
- `refundPayment()` → `/wallet/refund`

#### 2.1.3 Wallet Payment

**Flow:**
1. `walletApi.processPayment()` - Deduct from wallet
2. `ordersService.createOrder()` with `paymentMethod: 'wallet'`
3. Coins validated against wallet balance

**Coin System:**
- `nuqtaCoin` (global ReZ coins) - 100% of order value
- `promoCoin` (promotional coins) - 50% of order value max
- `storePromoCoin` (store-specific) - 30% of order value max

### 2.2 Authentication Flow

```
1. User enters phone number
2. POST /api/v1/auth/send-otp
   → OTP sent via SMS/WhatsApp
   → If has PIN → redirect to PIN login
3. POST /api/v1/auth/verify-otp
   → Validate OTP
   → If MFA enabled → return mfaRequired: true
   → Issue JWT tokens
4. Optional MFA verification
   → POST /api/v1/auth/mfa/verify-otp
   → TOTP validation
5. Token refresh
   → POST /api/v1/auth/refresh-token
```

**MFA Implementation:**
- TOTP-based (RFC 6238) using Node.js crypto
- 6-digit codes with 30-second window
- ±1 time-window tolerance for clock skew
- Backup codes support

### 2.3 Gateway Routing (nginx)

**Location Blocks:**
- `/api/search/*` → Search Service
- `/api/catalog/*` → Catalog Service
- `/api/auth/*` → Auth Service
- `/api/wallet/*` → Backend (legacy)
- `/api/orders/*` → Backend (legacy)
- `/api/store-visits/*` → Backend (legacy)

**Rate Limiting Zones:**
- `api_limit`: 50r/s global
- `auth_limit`: 100r/m for auth endpoints
- `merchant_limit`: 100r/s per merchant
- `pos_limit`: 30r/s for POS endpoints

getQueueStatus() → GET /store-visits/queue-status/:storeId

**Loyalty System:**

### 2.5 Bill Payment Flow (BBPS)

**Backend Routes:**
```typescript
GET  /types                    → Get bill types
GET  /providers               → Get providers by type
POST /fetch-bill              → Fetch bill amount
POST /pay                      → Pay bill via Razorpay
GET  /history                 → Payment history
POST /refund                  → Request refund
```

**Security:**
- Idempotency middleware (600s TTL)
- Financial rate limiting
- Customer number validation

### 2.6 Checkout Flow

```
1. Initialize checkout
   → Fetch cart from API
   → Load wallet coins
   → Fetch store details
   → Get user addresses
2. Apply promo codes
3. Toggle coin usage (nuqtaCoin, promoCoin, storePromoCoin)
4. Select fulfillment type (delivery, pickup, dine-in)
5. Select payment method
   → Wallet (deduct coins)
   → COD (create order, no payment)
   → Razorpay (open checkout)
6. Process payment
7. Create order
8. Navigate to success page
```
